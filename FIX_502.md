# Fix 502 Bad Gateway Error

## Problem
Render shows: "A web service has misconfigured its host and port. Bind your host to 0.0.0.0"

## Solution Applied

✅ **Updated `application.properties`** to bind to `0.0.0.0`:
```properties
server.address=0.0.0.0
server.port=${PORT:8081}
```

## What Changed

1. **Added `server.address=0.0.0.0`** - This makes Spring Boot listen on all network interfaces, not just localhost
2. **Kept `server.port=${PORT:8081}`** - Spring Boot will use Render's PORT env var (default 10000) or fallback to 8081

## Next Steps

1. **Commit and push the changes:**
   ```bash
   git add backend/src/main/resources/application.properties
   git commit -m "Fix: Bind Spring Boot to 0.0.0.0 for Render deployment"
   git push
   ```

2. **Render will automatically redeploy** when it detects the push

3. **Wait for deployment to complete** (usually 2-5 minutes)

4. **Check the service** - The 502 error should be resolved

## Verify It's Working

After redeployment, check:
- ✅ Service shows "Live" status
- ✅ No 502 errors
- ✅ Can access `https://your-backend-url.onrender.com/api/auth/me`

## If Still Having Issues

1. Check Render logs for any new errors
2. Verify environment variables are still set correctly
3. Check MongoDB connection (if you see database errors)

## Important Notes

- Render's default PORT is **10000**, not 8081
- Spring Boot will automatically use whatever PORT Render sets
- The `server.address=0.0.0.0` is **required** for Render to connect to your service
